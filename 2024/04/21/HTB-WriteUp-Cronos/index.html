0x00 Recon

這次除了常見的http 80, ssh 22 還出現了 dns 53
感覺53就很有問題，我們用nslookup看看對方ip
過程參考: https://book.hacktricks.xyz/network-services-pentesting/pentesting-dns

發現給了一個domain name cronos.htb
我們再利用看看dig DNS 他會不會吐出其他subdomaion

非常順利的拿到了一個叫 admin 的 subdomain
把 domain 和 ip 寫到 /etc/hosts 就可以正常訪問網頁了

0x01 SQLI to Login

在進到admin 後我們會看到這個簡陋的登入畫面

在經過觀察html 和 封包後大概確定這不是什麼CMS，是作者自己寫的登入頁面
這也意味著我們要自己打洞

但最後隨手甩了一個萬用密碼就進去了 笑死

0x02 Command Injection

登入後我們會看到這個畫面

這看起來就很有Command Injection的機會
直接上payload試試

我們就可以拿到reverseshell 了

0x03 Privilege Escalation

進入後發現自己只是www-html
經過基本的檢查發現沒什麼能利用的
因此使用linpeas 幫我們蒐集資訊

我們看到了這個有趣cron job
檔案在/var/www/ 我們可以去替換掉該檔案
我們先把/var/www/laravel/artisan先拿出來
在最上面加上 reverse shell payload

之後把修改後的檔案替換/var/www/laravel/artisan
然後靜待root執行我們的payload
這樣就拿到root了
我們可以在root下拿到root.txt，
